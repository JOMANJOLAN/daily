# ls 文件 / 目录查看

列出目录内容，最基础工具。

## 常用参数

```
-l  长格式显示（权限、属主、大小、时间）
-a  显示隐藏文件（以 . 开头）
-h  文件大小人性化显示（KB/MB/GB）
-t  按修改时间排序
-r  倒序
-d  只看目录本身，不看里面内容
```

## 常用组合

```bash
ls -l    # 详细列表
ls -lh   # 易读大小
ls -la   # 包含隐藏文件
ls -lrt  # 按时间倒序（找最近修改的文件）
```

# ps 进程查看

查看当前快照的进程信息。

## 常用用法

```bash
ps aux  # 查看系统所有进程（最常用）
ps -ef  # 同上，格式略有不同
ps -ef | grep process_name  # 过滤某个进程
```

## ps aux 字段说明

```
USER  用户
PID   进程 ID
%CPU  CPU占用
%MEM  内存占用
COMMAND  命令
```

# top 实时进程监控

## 常用操作

```
P  按 CPU 排序
M  按内存排序
T  按运行时间排序
k  输入 PID 杀死进程
q  退出
```

## 常用命令

```
top         # 直接打开
top -c      # 显示完整命令行
top -p PID  # 只监控某个进程
```

# lsof 查看文件 / 端口占用

list open files，Linux 一切皆文件，所以极强。

## 典型用法

```bash
lsof            # 所有打开文件
lsof -i         # 所有网络连接
lsof -i :port   # 查端口占用必备
lsof -p PID     # 某个进程打开了哪些文件
lsof file       # 谁在使用这个文件
```

# strace 系统调用追踪

跟踪进程内核调用，排查：

* 程序为什么卡住
* 打不开文件
* 连不上网络
* 权限问题

## 用法

```bash
strace -p PID   # 跟踪运行中的进程
strace command  # 直接启动并跟踪
strace -o log.txt command  # 输出到日志
strace -f       # 跟踪子进程
```

# gdb 进程调试神器

C/C++ 程序调试工具，定位崩溃、死循环、逻辑错误。

## 常用命令

```bash
gdb program      # 启动调试
run/r            # 运行程序
start            # 停在 main 开头
break/b line|fn  # 断点
next/n           # 单步跳过
step/s           # 单步进入
continue/c       # 继续运行
print/p var      # 打印变量
backtrace/bt     # 查看函数调用栈（崩溃必用）
quit/q           # 退出
```

## 调试 coredump（崩溃文件）

```bash
gdb program coredump
bt
```

# gcc C / C++ 编译器

把源码编译成可执行文件。

## 最基础编译

```bash
gcc test.c -o test  # 编译链接一步到位
```

## 常用参数

```
-c     只编译，不链接（生成 .o）
-g     带调试信息（给 gdb 用）
-O2    优化等级 2
-Wall  显示所有警告
-I     指定头文件路径
-L     指定库路径
-l     链接库
```

# make 自动构建

通过 Makefile 批量执行编译命令，避免重复敲 gcc。

## 最简单 Makefile 结构

**注意：命令行必须以 Tab 开头**

```
test: test.c
	gcc -g -Wall test.c -o test

clean:
	rm -f test
```

## 使用

```bash
make        # 编译
make clean  # 清理
```
