# 进程（Process）

操作系统分配资源的基本单位

## 运行状态

```
TASK_RUNNING          运行 / 就绪
TASK_INTERRUPTIBLE    可中断睡眠
TASK_UNINTERRUPTIBLE  不可中断睡眠
TASK_STOPPED          停止
TASK_ZOMBIE           僵尸
```

## 孤儿进程 & 僵尸进程

* 孤儿：父进程先退出，被 init(pid=1) 收养
* 僵尸：子进程退出，父进程未 wait，PCB 残留

# 信号

信号是进程间异步通知机制，用来通知进程发生了某事件

## 常见信号

```
SIGINT   2   Ctrl+C
SIGQUIT  3   Ctrl+\
SIGKILL  9   强杀（不能捕获、不能忽略）
SIGSEGV  11  段错误
SIGPIPE  13  管道读端关闭
SIGTERM  15  优雅终止
SIGCHLD  17  子进程退出
```

## 信号三种处理方式

* 默认动作：终止、忽略、停止、Core Dump
* 忽略
* 自定义捕获：注册信号处理函数

## 信号分类

* 可靠信号：34–64，支持排队
* 不可靠信号：1–31，可能丢失

## 信号与多线程

* 信号发给进程
* 但处理由某个线程执行
* 信号掩码是线程私有的

# 命令行

## 进程

```bash
ps aux               # 列出所有进程
ps aux | grep nginx  # 只看某个进程
top                  # 实时看进程（类似任务管理器）
pidof process_name   # 找进程
```

```bash
kill PID  # 杀死进程
kill -9 PID  # 强制杀死
pkill process_name # 按名字杀
killall process_name # 按名字杀
```

```bash
./program &  # 后台运行进程
nohup ./program & # 退出终端也不挂
```

```bash
pstree  # 看进程树
```

## 线程

```bash
ps -T -p PID  # 看一个进程的线程
top -H        # top里看线程
ps -o nlwp -p PID  # 看线程数
```

## 信号

```
常用信号：
1  SIGHUP  重启 / 重读配置
2  SIGINT  Ctrl+C
9  SIGKILL 强制杀死（不能忽略）
15 SIGTERM 正常退出（默认）
18 SIGCONT 继续
19 SIGSTOP 暂停
```

```bash
kill -signal_no PID
kill -15 PID  # 正常停止
kill -9 PID   # 强制杀死
kill -1 PID   # 让服务重读配置（不重启）
```

## 实用组合

```bash
kill -9 $(pidof nginx)      # 找到并杀死某个程序
top  # 看CPU占用
ps -T -p $(pidof firefox)   # 看某个程序的所有线程
nohup ./server > log.txt &  # 后台跑程序，退出终端不消失
```
