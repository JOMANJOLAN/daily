# 主干

1. 查VMA判权限、合法性
2. 逐级爬页表（页目录→页中间→PTE）
3. 缺页则判断：空页/fork共享/未分配/需要Swap/需要文件加载
4. 内存不足：看count、flags、LRU，决定回收/换出/回写

# 修正漏点

## 读写分动作
提前区分「写」会触发额外检查
- 读缺页：可以共享、可以走缓存、可以只读挂载
- 写缺页：立刻多一步：检查 `mapcount>1` → 直接触发 COW

---

## 爬页表
内核标准操作：
逐级页表遍历，中间页表不存在 → 先新建中间页表项，再往下找PTE
缺页表节点 → 先建节点，再找最终PTE

---

## 缺页分类
找到PTE，发现 `Present=0`，内核必查这5种：
1. 纯匿名新页：进程堆/栈从没分配过 → 直接分配zero页
2. fork共享模板页：`mapcount>1` + 写操作 → COW复制新页
3. 文件映射缺页：走page cache → 从磁盘读文件进page，挂PTE
4. Swap换回：PTE里存Swap槽 → 从Swap分区读回内存
5. 非法地址：直接SIGSEGV崩进程

---

## 内存不够时优先级
内存耗尽要腾内存：
1. 先看 `refcount/mapcount`：全0 → 直接回收干净页（最快）
2. 有引用/有映射：
   - 文件脏页 → 回写磁盘
   - 匿名页 → 走Swap换出
3. 全程靠 `flags(PG_swapbacked/PG_dirty)` + LRU链表挑选牺牲页

# 异常修复流程

1. 查VMA：地址合法？权限够？(读/写/执行匹配) → 非法直接崩
2. 逐级遍历页表：缺中间页表 → 新建；直到找到PTE
3. 看PTE Present：
   - 已存在物理页：直接放行（写命中还要检查COW）
   - Present=0：进入缺页异常
4. 缺页分支判断：
   - 新匿名页 → 分配zero page
   - 写共享fork页 → COW复制私有页
   - 文件映射 → 加载page cache、关联mapping+index
   - Swap标记 → 从Swap槽换回内存
5. 分配新物理页时内存不足：
   - 遍历LRU，看page flags、refcount、mapcount
   - 计数全0 → 直接回收
   - 脏文件页 → 回写磁盘
   - 匿名驻留页 → Swap换出
6. 腾出内存→分配新page→填充PTE→放行本次读写
