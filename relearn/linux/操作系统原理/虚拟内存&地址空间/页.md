# 页

```c
struct page {
    atomic_t _refcount;
    int mapcount;
    unsigned long flags;
    pgoff_t index;
    struct address_space *mapping;
    struct list_head lru;
};
```

## _refcount

记录当前有多少个内核实体正在使用这个物理页；

计数 > 0：页正在被使用，内核不能回收 / 释放该页；
计数 = 0：页无使用者，内核可以将其加入空闲链表，重新分配；

## mapcount

记录有多少个用户进程的虚拟地址空间，映射了这个物理页；
仅统计用户态映射，不统计内核态使用；
内核必须通过它判断页是否被多进程共享。

_refcount：管内核是否使用（全局引用）；
mapcount：管多少进程在用（用户态映射）；

## flags

用比特位存储页的各种状态 / 属性，一个字段承载多个信息（节省内存）；

```
PG_locked      页已加锁（正在读写，防止并发竞争）
PG_dirty       页是脏页（内容被修改，需要回写到磁盘）
PG_active      页在活跃 LRU 链表（近期被访问，暂不回收）
PG_slab        页用于 SLAB 内存分配器（内核小对象分配）
PG_reserved    页被保留（内核专用，不能分配给用户）
PG_anon        匿名页，无文件映射（栈、堆、mmap 匿名）
PG_swapcache   交换缓存页，已放入 swap 缓存，等待落交换分区
PG_swapbacked  依赖 Swap，内存不足可换出到磁盘 swap
```

## index

若当页是文件缓存页（页缓存）：
表示该页是文件的第几个页（文件偏移 = index × 页大小）；

若当页是匿名页（进程堆 / 栈）：
表示该页在进程虚拟内存区域（VMA）中的偏移索引；

## mapping

若指向文件地址空间：
该页是文件页缓存（存储文件内容，关联磁盘文件）；

若指向特殊匿名地址空间（PAGE_MAPPING_ANON）：
该页是匿名页（进程私有内存，无磁盘文件）；

内核通过它找到页的归属（哪个文件 / 哪个进程），是页回写、回收、查找的关键。

## lru

活跃 LRU 链表：近期被访问的页；
不活跃 LRU 链表：长期未访问的页；

内存不足时，内核从不活跃 LRU 链表回收最少使用的页，释放物理内存。

# 页表（Page Table）

```
pgd_t  Page Global Directory  全局页目录（顶层 = PML4）
pud_t  Page Upper Directory   上层页目录
pmd_t  Page Middle Directory  中间页目录
pte_t  Page Table Entry       页表项（最底层）
```

```c
typedef unsigned long pteval_t;

typedef struct { pteval_t pte; } pte_t;
typedef struct { pteval_t pmd; } pmd_t;
typedef struct { pteval_t pud; } pud_t;
typedef struct { pteval_t pgd; } pgd_t;
```

## 空页表级

```c
// IA-32架构的PUD/PMD简化（include/asm-x86/pgtable_32.h）
#define PTRS_PER_PUD    1  // PUD表仅1项，无实际意义
#define PTRS_PER_PMD    1  // PMD表仅1项，无实际意义

// PUD偏移计算（直接返回PGD表项）
static inline pud_t *pud_offset(pgd_t *pgd, unsigned long address) {
    return (pud_t *)pgd;
}

// PMD偏移计算（直接返回PUD表项）
static inline pmd_t *pmd_offset(pud_t *pud, unsigned long address) {
    return (pmd_t *)pud;
}
```

## pte

是内核与硬件沟通的渠道，比如COW时将RW设置为只读，缺页时将PRESENT设置为0，硬件只需要报错，然后由内核来处理

正常读写只会用到pte，发生异常时报告给内核

### flags

```
_PAGE_PRESENT   物理页在内存里置1，置0触发缺页异常
_PAGE_RW        可读置1，可读置0，用于COW
_PAGE_USER      内核态 / 用户态隔离
_PAGE_DIRTY     是否写过，写过必须落盘
_PAGE_ACCESSED  读 / 写命中，用于LRU回收
```

### 相关机制

当_PAGE_PRESENT=0时，pte的高位存放的是Swap 分区号和Swap 槽位偏移，缺页时直接找到对应磁盘位置

COW 靠 PTE：
PRESENT=1，RW=0，mapcount>1 → 写触发异常 → COW

Swap 靠 PTE：
PRESENT=0 → PTE 藏 Swap 入口 → 缺页换入

回收冷热靠 PTE：
ACCESSED 统计热度 → LRU 选牺牲页

脏页落盘靠 PTE：
DIRTY 证明改过 → 必须写回文件 / 写 Swap
