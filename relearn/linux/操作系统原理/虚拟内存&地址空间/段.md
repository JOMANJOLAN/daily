# 用户空间

## 范围

```
0x0000000000400000  x86_64 典型起始
0x00007fffffffffff  用户空间上限
```

## .text（代码段）

存放程序二进制指令、函数体
属性：只读、可执行，多进程可共享
编译时确定大小，运行时不可修改

## .rodata（只读数据段）

存放字符串常量、const 全局 / 静态常量
属性：只读，防止意外修改

## .data（数据段）

存放已初始化的全局变量、静态变量（非零初始值）
属性：读写，大小在编译时确定

## .bss（未初始化数据段）

存放未初始化 / 初始化为 0 的全局变量、静态变量
不占用可执行文件空间，运行时由系统自动清零

## heap（堆）

动态内存分配区（malloc/new）
地址向上增长，需手动释放（否则泄漏）

## mmap（内存映射段）

加载动态链接库（`libc.so`）、文件映射、匿名映射
位于堆与栈之间，地址通常向下增长

## stack（栈）

存放局部变量、函数参数、返回地址、栈帧
地址向下增长，自动分配 / 释放，有大小限制（易溢出）

# VMA

```c
struct vm_area_struct {
    unsigned long vm_start;
    unsigned long vm_end;
    struct mm_struct *vm_mm;
    pgprot_t vm_page_prot;
    unsigned long vm_flags;
    struct file *vm_file;
    const struct vm_operations_struct *vm_ops;
    struct rb_node vm_rb;
};
```

## vm_start & vm_end

精准标记这段 VMA 占用的虚拟地址空间，内核通过这两个值判断虚拟地址属于哪段内存

## vm_mm

mm_struct 是进程整个虚拟地址空间的总控结构，包含页表、所有 VMA 的链表 / 红黑树

标记这段 VMA 属于哪个进程
所有属于同一个进程的 VMA，vm_mm 都指向同一个 mm_struct

## vm_page_prot

页表项的硬件权限属性（CPU 硬件识别的权限）

包含权限：读（R）、写（W）、执行（X）、缓存模式、特权级等

当这段虚拟内存映射到物理页时，内核会把这个权限写入页表，CPU 硬件直接校验

vm_flags 是内核逻辑权限，这个是硬件实际权限，内核会根据 vm_flags 计算出 vm_page_prot

## vm_flags

内核用它判断 VMA 类型、处理缺页异常、权限检查

```
VM_READ     可读
VM_WRITE    可写
VM_EXEC     可执行
VM_SHARED   共享映射（多进程共享同一段物理内存）
VM_PRIVATE  私有映射（写时复制）
VM_ANON     匿名映射（无文件关联，比如堆、栈）
VM_STACK    栈区域
```

## vm_file

指向被映射的文件结构体
文件映射 mmap（比如把可执行文件、库文件映射到进程内存）

为空的情况：
匿名映射（堆、栈、匿名 mmap）
无文件关联的内存区域

缺页时，内核通过这个指针从文件读取数据到物理内存

## vm_ops

```c
struct vm_operations_struct {
    // 打开 VMA（创建/引用时调用）
    void (*open)(struct vm_area_struct *vma);
    // 关闭 VMA（释放时调用）
    void (*close)(struct vm_area_struct *vma);
    // 【核心】缺页异常处理（用户访问未映射页时调用）
    int (*fault)(struct vm_fault *vmf);
    // 预缺页（提前加载内存，优化性能）
    int (*page_mkwrite)(struct vm_fault *vmf);
    // 写时复制（COW）缺页处理
    int (*pfn_mkwrite)(struct vm_fault *vmf);
    // 拆分 VMA 时调用
    void (*split)(struct vm_area_struct *vma);
    // 脏页同步到文件（msync 调用）
    int (*page_mkwrite)(struct vm_fault *vmf);
    // 读取特殊 VMA 数据
    ssize_t (*read)(struct vm_area_struct *vma, char __user *buf, size_t len, loff_t *ppos);
    // 写入特殊 VMA 数据
    ssize_t (*write)(struct vm_area_struct *vma, const char __user *buf, size_t len, loff_t *ppos);
};
```

## vm_rb

嵌入 VMA 中的红黑树节点

进程的所有 VMA 组织在红黑树中
快速根据虚拟地址查找对应的 VMA（时间复杂度 O (log n)）

内核同时用链表 + 红黑树管理 VMA：
链表：遍历所有 VMA
红黑树：快速地址查找
