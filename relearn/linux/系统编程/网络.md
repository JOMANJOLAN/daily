# 通信

服务端（被动等待连接）
socket()，bind()，listen()，accept()，读写数据，close()

客户端（主动发起连接）
socket()，connect()，读写数据，close()

---

## socket

创建一个网络通信的文件描述符，指定协议族、类型、协议

```c
#include <sys/socket.h>

// 成功返回文件描述符sockfd，失败返回-1
int socket(int domain, int type, int protocol);
```

* domain：协议族
* type：套接字类型
* protocol：协议（填 0，自动匹配）

---

## bind

将 socket 与本机 IP 地址 + 端口号绑定，服务端必须绑定，客户端无需绑定（系统自动分配）

```c
#include <sys/socket.h>

// 成功返回0，失败返回-1
int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```

* 端口号：1024 以下是系统保留端口，自定义用 1024~65535
* 字节序：用htons()将主机序转化为网络序（大端序）
* IP 地址：INADDR_ANY 表示监听本机所有网卡 IP（最常用），用htonl转换

---

## listen

将 socket 设置为被动监听模式，内核创建连接队列，等待客户端连接

TCP 是三次握手，握手过程中连接会放入内核队列，listen就是开启这个队列

```c
#include <sys/socket.h>

// 成功返回0，失败返回-1
int listen(int sockfd, int backlog);
```

* backlog：半连接队列 + 全连接队列的最大长度（内核限制，一般填 5、10、128）

---

## accept

从全连接队列中取出一个已完成握手的客户端连接，返回新的文件描述符（专门用于和该客户端通信）

```c
#include <sys/socket.h>

// 成功返回客户端通信fd，失败返回-1；阻塞式等待
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```

* 原sockfd：只负责监听，不负责通信
* 返回值client_fd：专属通信 fd，服务端用它和客户端收发数据
* 默认阻塞：没有客户端连接时，进程会挂起等待

---

## connect

客户端专用，主动向服务端发起 TCP 三次握手，建立连接

```c
#include <sys/socket.h>

// 成功返回0，失败返回-1
int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```

---

## inet_pton

把人类可读的 IP 字符串（如 "192.168.1.1"）转换成 网络能识别的二进制整数 的函数，专门给 sockaddr_in 赋值 IP 地址用

```c
#include <arpa/inet.h>  // 必须包含这个头文件

// 函数定义
int inet_pton(int af, const char *src, void *dst);
```

* af：地址族，AF_INET (IPv4)
* src：源：字符串格式 IP，"127.0.0.1"
* dst：存放转换后的二进制，&结构体.sin_addr

## 网络地址表述

```c
#include <sys/socket.h>

// 通用套接字地址结构（所有网络函数统一接收的类型）
struct sockaddr {
    sa_family_t     sa_family;      // 地址族（AF_INET/AF_INET6等）
    char            sa_data[14];    // 协议地址（IP+端口，直接用极难）
};
```

```c
#include <netinet/in.h>

struct sockaddr_in {
    sa_family_t    sin_family;  // 协议族（AF_INET）
    uint16_t       sin_port;    // 端口号（必须转网络字节序）
    struct in_addr sin_addr;    // IP地址
    char           sin_zero[8];// 填充位
};
```

```c
#include <netinet/in.h>

// 纯 IPv4 地址结构体（只存一个IP）
struct in_addr {
    uint32_t    s_addr;    // 32位IPv4地址（网络字节序）
};
```

```c
#include <sys/socket.h>

typedef unsigned short int sa_family_t;
```

```c
#include <sys/socket.h>

typedef unsigned int socklen_t;
```

# epoll

Linux 内核提供的IO 多路复用神器，单线程可以同时监听成千上万个文件描述符，只处理活跃事件，无事件时休眠，CPU 占用极低

---

## 创建 epoll 实例

```c
#include <sys/epoll.h>

// 成功返回epoll文件描述符epfd，失败返回-1
int epoll_create(int size);
```

* size：内核初始化事件大小（Linux 2.6.8+ 已忽略，填任意正整数即可）

---

## 增 / 删 / 改 监听事件

```c
#include <sys/epoll.h>

// 成功返回0，失败返回-1
int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);
```

```c
#include <sys/epoll.h>

struct epoll_event {
    uint32_t events; // 监听事件：EPOLLIN(读)、EPOLLOUT(写)、EPOLLET(边缘触发)
    epoll_data_t data; // 存储用户数据（一般存fd）
};
```

* op：操作类型，EPOLL_CTL_ADD、EPOLL_CTL_DEL、EPOLL_CTL_MOD
* event：监听事件结构体

---

## 等待事件发生（核心）

```c
#include <sys/epoll.h>

// 返回活跃事件数量，0=超时，-1=错误
int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout);
```

* 阻塞等待，直到有活跃 fd（可读 / 可写），内核将活跃事件填入events数组
* 单线程循环处理所有活跃连接，实现高并发

---

## 触发模式

### 水平触发 LT（默认）

只要缓冲区有数据，就一直触发（简单、安全）

### 边缘触发 ET

只有状态变化时触发一次（高效，必须配合非阻塞 IO）
