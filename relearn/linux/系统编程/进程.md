# 核心概念

Linux 中进程是程序的运行实例，每个进程有唯一的 PID（进程 ID），由内核管理。所有进程都源自 init（PID=1），形成树形结构

---

# fork 创建子进程

```c
#include <unistd.h>

pid_t fork(void);
```

* 复制当前进程创建子进程，写时复制（COW）
* 子进程会继承父进程的：内存、寄存器、文件描述符、环境变量、信号处理（除了 SIGCHLD）、工作目录等
* 父子进程的执行顺序由内核调度器决定（不确定）。
* 内核会为子进程分配新的 PID，子进程的 PPID（父进程 ID）指向原进程
* fork() 有两个返回值：
  * 父进程中：返回子进程的 PID（正数）
  * 子进程中：返回 0
  * 出错时：返回 -1（如资源不足）

# exec 替换进程镜像

```c
#include <unistd>

extern char **environ; // 全局变量，存储当前进程的环境变量

int execl(const char *path, const char *arg, ... /* (char  *) NULL */);
int execlp(const char *file, const char *arg, ... /* (char  *) NULL */);
int execle(const char *path, const char *arg, ... /*, (char *) NULL, char * const envp[] */);
int execv(const char *path, char *const argv[]);
int execvp(const char *file, char *const argv[]);
int execve(const char *path, char *const argv[], char *const envp[]);
```

* exec() 会用新程序替换当前进程的代码段、数据段、堆栈，但PID 不变（只是进程内容被替换）
* exec 成功时：无返回值（因为进程已被替换）；exec 失败时：返回 -1，并设置 errno（如路径错误、权限不足）；
* 常用场景：shell 执行命令（比如你在终端输入 ls，shell 会 fork 子进程，再 exec 执行 ls）。

# wait / waitpid 父进程回收子进程

```c
#include <sys/wait.h>

// wstatus：指针，用于存储子进程的退出状态
// 可传 NULL 表示不关心退出状态
pid_t wait(int *wstatus);
// options：可选标志，设置非阻塞等模式（可组合使用，用 |）
pid_t waitpid(pid_t pid, int *wstatus, int options);
```

```
WIFEXITED(w)    判断子进程是否正常退出（exit/_exit）
WEXITSTATUS(w)  获取正常退出的状态码（0-255）
WIFSIGNALED(w)  判断子进程是否被信号终止（如 kill -9）
WTERMSIG(w)     获取终止子进程的信号号
WIFSTOPPED(w)   判断子进程是否被暂停
WSTOPSIG(w)     获取暂停子进程的信号号
```

* 如果子进程结束但父进程未回收，子进程会变成 “僵尸进程（Zombie）”：PID 仍存在，占用系统资源，直到父进程回收或父进程退出
* wait() / waitpid() 就是父进程等待子进程终止，并回收其资源（释放 PID、获取退出状态）
* wait() 只能等待任意子进程
* waitpid() 可以指定等待某个子进程，还能设置非阻塞，非阻塞时若无子进程结束就立刻返回 0

# exit 进程终止

```c
#include <stdlib.h>

void exit(int status);
```

```c
#include <unistd.h>

void _exit(int status);
```

* exit 是进程主动终止的系统调用
* 执行注册的清理函数（atexit() 注册的）
* 关闭所有打开的文件描述符
* 向父进程发送 SIGCHLD 信号
* 释放资源，将进程状态设为「僵尸态」，等待父进程 wait() 回收

# 进程关系

## 父子进程

* 由 fork() 创建，子进程 PPID 指向父进程 PID
* 父进程可以通过 wait() 回收子进程，子进程结束会给父进程发 SIGCHLD
* 父进程先退出，子进程会变成孤儿进程

## 兄弟进程
同一父进程 fork 出的多个子进程互为兄弟，彼此无直接关联（除非通过管道 / 信号等通信）

## 僵尸进程（Zombie）

* 子进程结束，但父进程未调用 wait() 回收；
* ps -ef 中状态为 Z，PID 存在，占用内核资源；

**解决方法**

* 父进程调用 wait()/waitpid() 回收
* 父进程退出，孤儿进程被 init 接管并回收
* 给父进程发 SIGCHLD 信号（内核自动处理）

## 孤儿进程

* 父进程先于子进程退出，子进程无父进程；
* 被 init 进程（PID=1）收养，PPID 变为 1；
* 无危害（init 会自动 wait() 回收，不会变成僵尸）。
